## 1. 输入模式（三选一，互斥）

`show_info()` 起始处（≈第 1898 行）依据三组输入框谁非空，分发到三条计算路径，**没有混合输入的兼容逻辑**。任一模式必须把对应五个输入框全部填齐，否则弹警告框。

| 模式 | 输入框 | 计算行为 |
|---|---|---|
| **公历** | 年、月、日、时(0–23)、分 | 阳历 → 农历（`ZhDate.from_datetime`）→ 算节气、四柱、元、定局 |
| **阴历**（实为「干支直输」）| 年干支、月干支、日干支、时干支、节气名 | **不重新换算时间**，节气与四柱直接采用用户输入；只用时干支去算旬头、起局 |
| **农历** | 农历年、农历月、农历日、时(0–23)、分(0–59) | 农历 → 公历（`ZhDate.to_datetime`）→ 同公历路径 |

> 命名陷阱：在「农历」路径中变量 `jieqi_nongli` 实际存的是**分钟**（`minutee = jieqi_nongli`），属历史命名遗留，无功能问题。

## 2. 节气计算（公历 / 农历路径）

### 2.1 算法
- 使用 `ephem` 库计算太阳视黄经（`Sun → Equatorial → Ecliptic`）。
- 给每个节气时间统一**加 1/3 天 = +8 小时**作为时区/调整偏移（`datetime.timedelta(days=1/3)`），随后只保留到「小时」精度（分秒丢弃）。
- 一次性生成连续 26 个节气时间点 `b[0..25]`。

### 2.2 当日所属节气判定
**用 `v = datetime.datetime(year, month, day, 23, 59, 0)`**，不是用户输入的 hour/minute。
含义：以输入日的「23:59」为查询点，落在 `b[i] ≤ v ≤ b[i+1]` 区间，取 `jieqi_list[i]`。即**当日跨节气时，统一按当日 23:59 时刻所处节气取值**。

> 该 23:59 约定与 docx 草稿无矛盾，是程序实际行为。

### 2.3 节气名映射表（`jieqi_list`，索引 0 起）

## 3. 四柱计算

### 3.1 年干支（公历/农历路径，会受节气影响修正）

1. 取输入公历年 `year` 作初值。
2. 在节气表 `jieqinianganjisuanlist = [小寒, 大寒, 立春, ... 冬至]`（**注意起点是小寒**）中找 `JQ` 的 index。
3. 修正规则（**程序写法**）：
   - `index ≤ 1`（小寒、大寒）→ `year = year - 1`
   - `index == 23`（冬至）且 `month == 1` → `year = year - 1`
   - 其它 → 不变
4. **效果**：年干支于「**立春**」交界翻新年——立春之前（含小寒、大寒）算上一年；立春当天起算新一年。这与 docx 草稿的"按节气定年"一致。

### 3.2 年干支换算（基准）
- 基准：公元 0 年（理论上）对应索引 0，公式：
  - `天干index = (year_eff - 4) % 10`，`地支index = (year_eff - 4) % 12`
  - `Tiangan = [甲乙丙丁戊己庚辛壬癸]`，`Dizhi = [子丑寅卯辰巳午未申酉戌亥]`
- 验证：`year_eff = 1900` → 庚(6)子(0) = 庚子 ✓

### 3.3 月干（节气月，由 `yuetiangangengxin` 输出）

1. 在 `JQLIEBIAO = [立春, 雨水, 惊蛰, 春分, ...大寒]`（起点立春）中找 `JQ` 的 index `i`，再求月号：
   - `i % 2 == 0`（节）：`shijianyuegengxin = (i + 2) // 2`
   - `i % 2 == 1`（中气）：`shijianyuegengxin = (i + 1) // 2`
   - 含义：立春/雨水 → 1（寅月）；惊蛰/春分 → 2（卯月）……依节气月分组。
2. 月干按"五虎遁年起月"分组（**程序写法**）：

| 年干 | 寅(1)→月干序列 |
|---|---|
| 甲、己 | 丙丁戊己庚辛壬癸甲乙 |
| 乙、庚 | 戊己庚辛壬癸甲乙丙丁 |
| 丙、辛 | 庚辛壬癸甲乙丙丁戊己 |
| 丁、壬 | 壬癸甲乙丙丁戊己庚辛 |
| 戊、癸 | 甲乙丙丁戊己庚辛壬癸 |

公式：`月干 = tgNyue[(shijianyuegengxin - 1) % 10]`

### 3.4 月支

固定从寅月开始：`Yuedizhi = [寅, 卯, 辰, 巳, 午, 未, 申, 酉, 戌, 亥, 子, 丑]`；`月支 = Yuedizhi[shijianyuegengxin - 1]`。

> ⚠ 程序中还有一处早期写法 `yuetiangan = tg1yue[(shijianyue - 1) % 10]`，使用的是**农历月** `shijianyue`，但该结果不进入最终显示，只有 `yuetiangangengxin / yuedizhigengxin`（节气月）会输出到界面。

### 3.5 日干支（公历/农历路径）

1. **元旦干支基准**（函数 `calculate_ganzhiyuandan(year)`）：1900-01-01 = 甲(0)戌(10)；从 1900 起，**平年加 5，闰年加 6**，累计偏移到目标 `year` 即得当年元旦干支。
2. **当日干支**（基于元旦干支 + 月内偏移表）：
   
   ```
   tiangan_index = [-1, 0, -2, -1, -1, 0, 0, 1, 2, 2, 3, 3]   # 月偏移（天干）
   dizhi_index   = [-1, 6, 10, 5, -1, 6, 0, 7, 2, 8, 3, 9]    # 月偏移（地支）
   ```
3. **闰年补正**：若 `year` 是闰年且 `month >= 3`，结果索引各 `+1`（跨过 2 月 29 日）。
4. 最终 `ritiangan = Tiangan[ritiangan_index - 1]`，`ridizhi = Dizhi[ridizhi_index - 1]`（程序里这个 `-1` 是巧设的偏移，整套表为闭包公式，未引入第三方农历库）。

### 3.6 时支（小时→地支）

```
0    时 → 子
1–2  时 → 丑
3–4  时 → 寅
5–6  时 → 卯
7–8  时 → 辰
9–10 时 → 巳
11–12时 → 午
13–14时 → 未
15–16时 → 申
17–18时 → 酉
19–20时 → 戌
21–22时 → 亥
23   时 → 子
```

> 关键：**23:00 与 0:00 都归子时，但日干支不在 23:00 翻日**——本程序采用"零点换日"的简化派系，**没有早子时/夜子时之分**。

### 3.7 时干（五鼠遁日起时）

| 日干 | 子(p=0)→时干序列 |
|---|---|
| 甲、己 | 甲乙丙丁戊己庚辛壬癸 |
| 乙、庚 | 丙丁戊己庚辛壬癸甲乙 |
| 丙、辛 | 戊己庚辛壬癸甲乙丙丁 |
| 丁、壬 | 庚辛壬癸甲乙丙丁戊己 |
| 戊、癸 | 壬癸甲乙丙丁戊己庚辛 |

公式：`时干 = tgNshicheng[p % 10]`，其中 `p` 为 `shichengdizhi` 在 `Dizhi` 中的索引（0=子, 1=丑, …）。

## 9. 三奇六仪布天盘（旬头移位）

代码用一个 8 元数组 `sanqiliuyizhuangdonglist` 实现"**旬头宫 → 当前时干宫 的旋转**"：

1. 从 `xuntougonghaozhuangdong` 起，沿 8 宫转动序**顺时针**读取 8 个宫 `[3]` 位置（即地盘三奇六仪）的文本，得到长度为 8 的数组（遇到坤二位置 `i%8 == 5` 时读取双行文本——因坤二宫包含中五寄宫数据）。
2. 从 `sanqiliuyilaodagonghaozhuandong` 起，沿 8 宫转动序顺时针写到各宫 `[4]` 位置（天盘）。
3. 在写入宫位置 `[5]` 处显示 `+`。
4. 若写入的文本长度 ∈ {5, 7}（双行文本含 `\n`），则以 `setPlainText` 写入并允许多行排版；若长度 ∈ {1, 3}（单行），则以 `border:none;padding-top:10px` 样式置入。

> 中五位置永远不会被天盘直接覆盖：所有原"中五的三奇六仪"都通过"坤二寄宫"机制进入数组，再被旋转到目标位置。

## 10. 八神

### 10.1 顺序数组
- 阳遁：`BASHEN = [值符, 螣蛇, 太阴, 六合, 白虎, 玄武, 九地, 九天]`
- 阴遁：`BASHENNI = [值符, 九天, 九地, 玄武, 白虎, 六合, 太阴, 螣蛇]`（在代码中即 `BASHEN[0] + BASHEN[-1..-7]`）

### 10.2 布列方式（不论阴阳遁，**写入方向都按 8 宫转动序顺时针**）
```python
for i in range(start, start + 8):
    output[dingweigongzhuangdong[i % 8][0]] = BASHEN[i - start]  # 阳遁
                                              BASHENNI[i - start] # 阴遁
```
其中 `start = sanqiliuyilaodagonghaozhuandong`。即"值符"放在「**当前时干所在宫（中五时寄到坤二）**」的 `[0]` 位置，其它七神顺时针排列。

> **阴阳遁差异只表现为元素顺序的反转**（"九天"在阳遁是末位、阴遁是第二位），方向都是顺时针。这一点 docx 草稿写法略简化，程序为准。

### 10.3 四种边界情形分支（程序中实际写法）
| 情形 | xuntougonghaozhuangdong 取自 | sanqiliuyilaodagonghaozhuandong 取自 |
|---|---|---|
| ① 旬头中五 + 时干中五 | 坤二位 (=5) | 坤二位 (=5) |
| ② 旬头中五 + 时干非中五 | 坤二位 (=5) | 时干实际宫 |
| ③ 旬头非中五 + 时干中五 | 旬头实际宫 | 坤二位 (=5) |
| ④ 都不在中五 | 各自实际宫 | 各自实际宫 |

四种分支的 `BASHEN/BASHENNI` 主写入循环代码完全等价，仅起始下标不同。

## 11. 九星

### 11.1 顺序数组（含原始落宫）
```
JUIXING = [天蓬, 天芮, 天冲, 天辅, 天禽, 天心, 天柱, 天任, 天英]
对应宫号：0=坎一  1=坤二  2=震三  3=巽四  4=中五  5=乾六  6=兑七  7=艮八  8=离九
```

### 11.2 布列规则（不分阴阳遁，**始终顺 9 宫序号写入**）
```python
# 从旬头宫起，截取 9 颗星的"该宫起始顺序"
JUIXINGSHURU = [ JUIXING[(xuntougonghao + k) % 9] for k in range(9) ]

# 从"当前时干落宫"起，顺序写入 9 宫
for k in range(9):
    target_gong = (sanqiliuyilaodagonghao + k) % 9
    if target_gong == 4:
        写到中宫 [0]   # 中宫只有 [0] [1] 两位，此处用 [0]
    else:
        写到 dingweigong[target_gong][2]
```
含义：值符把旬头宫位上的九星带到当前时干所在宫，**九星整体按 9 宫顺序滚动**，遇中五则居中。

## 12. 八门

### 12.1 顺序与原始落宫
```
BAMEN = [休, 生, 伤, 杜, 景, 死, 惊, 开]
对应 8 宫转动序：0=坎一  1=艮八  2=震三  3=巽四  4=离九  5=坤二  6=兑七  7=乾六
```

### 12.2 计算时干到旬头的差值
- `DizhiZ`：当前 `xuntouzi` 所属那张 10 元旬表（如 `list1jiazi`）。
- `shichengdizhiindex`：当前 `shichengganzhi` 在 `DizhiZ` 中的索引（0–9）。
- `xuntouzi1index`：`xuntouzi` 在 `DizhiZ` 中的索引（恒为 **0**）。
- **`shichengchazhi = |shichengdizhiindex - xuntouzi1index|`**（实际就是 `shichengdizhiindex`）。

### 12.3 值使飞宫
飞行**在 9 宫序号上 mod 9 循环**（不是 8 宫转动序）：
- 阳遁：`zhishiweizhifeigongindex = (xuntougonghao + shichengchazhi) % 9`
- 阴遁：`zhishiweizhifeigongindex = xuntougonghao - shichengchazhi`；若 < 0 再 `+ 9`。

若飞到中五（`= 4`），显示位置寄到坤二（在转动序中下标 = 5）。

> 🔍 **与 docx 草稿的差异**：docx 写道"旬头若在中五则从坤二宫开始飞"。程序实际是**从中五本宫开始（按 9 宫序循环）飞**，并不切换起点到坤二，仅在终点落到中五时改寄坤二显示。用户要求以程序为准 → **以本规则为准**。

### 12.4 值使是谁
- 旬头**不在中五**：值使 = 旬头宫的八门，做法是把 `BAMEN[xuntougonghaozhuangdong2]`（旬头宫在转动序中的下标）末尾追加 `(使)`。
- 旬头**在中五**：值使 = 死门，做法是把 `BAMEN[5]`（即"死"）追加 `(使)`。

### 12.5 写入八门
不分阴阳遁，写入方向都是 8 宫转动序的顺时针：
```python
# 旬头不在中五（普通情形）
zhishidingweilist = [ BAMEN[(xuntougonghaozhuangdong2 + k) % 8] for k in range(8) ]
for k in range(8):
    target = (zhishidingweizhuangdongindex + k) % 8
    output[dingweigongzhuangdong[target][1]] = zhishidingweilist[k]

# 旬头在中五时改用：起始下标固定为 5（即"死(使)"），其余同上
zhishidingweilist = [ BAMEN[(5 + k) % 8] for k in range(8) ]
```
亦即：**把"值使"那张门放在 `zhishiweizhifeigongindex` 所对应的宫位（中五寄坤二），其余七门顺时针展开**。

## 13. 整体流程汇总

```
输入(公历 / 阴历 / 农历) ─┐
                          ├→ 解析为四柱 + 节气 (公历/农历路径) 或直采 (阴历路径)
                          │
                          ├→ 依日干支 → 元日（上/中/下元）
                          │
                          ├→ 依节气   → 阴阳遁
                          │
                          ├→ 起局     → 起局宫 dingweigong_index
                          │
                          ├→ 三奇六仪入九宫 (阳顺/阴逆，中五寄坤二显示)
                          │
                          ├→ 依时干支求 旬头(xuntou) + 旬头宫(xuntougonghao)
                          ├→ 依时干求   sanqiliuyilaoda + sanqiliuyilaodagonghao
                          │
                          ├→ 三奇六仪天盘 = 8 宫转动序旋转，旬头→当前时干宫
                          ├→ 八神 = 阳/阴遁列表，从 sanqiliuyilaodagonghao(寄)起，顺时针写 8 位
                          ├→ 九星 = 9 宫序滚动，旬头处的星 → 当前时干宫，其余依次
                          ├→ 八门 = 旬头宫八门 + (使)，按 9 宫序飞到值使宫（中五寄坤二），
                          │        以值使宫为起点顺时针写 8 门（不分阴阳）
                          │
                          └→ 渲染九宫盘
```
